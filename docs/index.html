🏁 Getting Started

Minimal code to start a scheduled task:

static void Main()
{
    var scheduler = new SchedulerEngine(() =>
    {
        Console.WriteLine("Scheduled task executed!");
    });

    scheduler.Start();
}

⚙️ Configuration

Add this section in your App.config or Web.config:

<configuration>
  <configSections>
    <section name="schedulerSettings" type="AutomateCore.Configuration.SchedulerConfigSection, AutomateCore" requirePermission="false" />
  </configSections>

  <schedulerSettings>
    <add scheduleType="Hourly" enabled="true" hour="0" minute="0" />
  </schedulerSettings>
</configuration>

🗓️ Schedule Configuration Options

Attribute Description
scheduleTypeImmediate, Daily, Weekly, Monthly, etc.
enabledtrue / false
hourHour for Daily/Weekly/Monthly schedules
minuteMinute for execution
dayOfWeekDay(s) for Weekly
dayOfMonthDay for Monthly

🧠 Supported ScheduleType Values

  • Immediate — Executes immediately on startup
  • EveryXMinutes — 1, 2, 5, 10, 15, 30 minutes
  • EveryXHours — 2, 4, 6 hours
  • Hourly — At specified minute each hour
  • Daily, Weekly, Monthly
  • TwiceDaily, Weekends, Quarterly, etc.

🧑‍💻 Usage & Lifecycle Callbacks

// Declare a field
private ISchedulerEngine _scheduler;

// Main task
void RunTask() => Console.WriteLine("Running scheduled task...");

// Callbacks
void NotifyServiceStarted(DateTime t) => Console.WriteLine($"✅ Started at {t}");
void NotifyServiceCompleted(DateTime t) => Console.WriteLine($"✅ Completed at {t}");
void NotifyServiceSkipped(string reason, DateTime t) => Console.WriteLine($"⚠️ Skipped at {t} — {reason}");
void NotifyServiceFailed(Exception ex, DateTime t) => Console.WriteLine($"❌ Failed at {t} — {ex.Message}");

// Initialize
_scheduler = new SchedulerEngine(
    RunTask,
    onTaskStarted: NotifyServiceStarted,
    onTaskCompleted: NotifyServiceCompleted,
    onTaskSkipped: NotifyServiceSkipped,
    onTaskFailed: NotifyServiceFailed
);
