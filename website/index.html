Add your task to the scheduler with one line of code

TaskImplementation.cs 
using AutomateCore;
using System.Threading;
using System.Threading.Tasks;

namespace YourNamespace
{
    public class SampleTask : IScheduledTask
    {
        private readonly ILogger<SampleTask> _logger;
        
        // Constructor injection supported
        public SampleTask(ILogger<SampleTask> logger) {
            _logger = logger;
        }
        
        public string TaskName => "SampleTask";
        
        public async Task ExecuteAsync(CancellationToken ct)
        {
            // Your business logic here
            await ProcessData();
            
            // Log completion
            _logger.LogInformation("Task completed");
        }
        
        public Task OnErrorAsync(Exception ex)
        {
            // Custom error handling
            _logger.LogError(ex, "Task failed");
            return Task.CompletedTask;
        }
    }
}

Pro Tip

Use constructor injection to access services like ILogger and IConfiguration in your tasks. AutomateCore supports full DI integration.

FAQ

Frequently Asked Questions

Common questions about AutomateCore

Is AutomateCore suitable for high-frequency tasks?

Yes, AutomateCore is optimized for both high-frequency (minute-level) and long-interval scheduling. The lightweight core ensures minimal overhead even for tasks running every few minutes.

Can I migrate from Windows Task Scheduler?

Absolutely. AutomateCore provides migration tools to convert your existing Windows Task Scheduler jobs into AutomateCore configurations with minimal effort.

Does it support distributed environments?

While AutomateCore is designed as a Windows Service, it can be configured to work in distributed environments using shared storage for task coordination.

What's the performance overhead?

The core scheduler adds less than 1% CPU overhead in typical usage. Memory footprint is minimal (~5MB) and scales linearly with the number of active tasks.
