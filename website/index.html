IMPLEMENTATION

Simple Integration

With just a few lines of code, you can have a fully functional scheduled task running in your Windows service.

  • Implement the IScheduledTask interface
  • Configure your schedule in XML
  • Register with the scheduler service
TaskImplementation.cs 
using AutomateCore;
using System.Threading;
using System.Threading.Tasks;

namespace YourNamespace
{
    public class SampleTask : IScheduledTask
    {
        public string TaskName => "SampleTask";
        
        public async Task ExecuteAsync(CancellationToken ct)
        {
            // Your business logic here
            await ProcessData();
            
            // Log completion
            return Task.CompletedTask;
        }
        
        public Task OnErrorAsync(Exception ex)
        {
            // Custom error handling
            LogError(ex);
            return Task.CompletedTask;
        }
    }
}
CONFIGURATION

Declarative Scheduling

Define your schedules in clean XML without touching code

Schedule Types

Interval-Based

Run tasks at fixed intervals (minutes, hours)

Time-Based

Daily, weekly, monthly at specific times

Custom Patterns

Weekends, business days, quarters

Immediate

Run once on service startup

App.config 
<configuration>
  <configSections>
    <section name="scheduledTasks" 
             type="AutomateCore.Scheduling.ScheduledTaskSection, AutomateCore"/>
  </configSections>

  <scheduledTasks>
    <task name="DataSyncTask" 
          type="YourNamespace.DataSyncTask, YourAssembly"
          scheduleType="Daily" 
          runAt="14:30" 
          retryOnFailure="true"
          maxRetries="3"/>
          
    <task name="CleanupTask"
          type="YourNamespace.CleanupTask, YourAssembly"
          scheduleType="EveryXMinutes"
          interval="15"/>
  </scheduledTasks>
</configuration>

Configuration Tips

  • Use scheduleType="Daily" for once-daily tasks
  • EveryXMinutes is perfect for frequent background jobs
  • Set retryOnFailure="true" for resilient operations
