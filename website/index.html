About AutomateCore Scheduler

AutomateCore is a lightweight, config-first task scheduler built for .NET Windows Services. It was designed to reduce boilerplate and offer a practical, testable foundation for background automation.

💡 Project Background

I created AutomateCore while working on Windows services that required multiple recurring tasks. Instead of rewriting scheduling logic each time, I built a reusable core with clean configuration and interface-based task definitions. Eventually, I decided to open-source it for anyone facing similar needs.

🔧 Built For Developers

This isn't a framework trying to be everything — it's a focused utility for developers who prefer keeping their task logic in code and schedules in config. Whether you're automating syncs, reports, or timed jobs, AutomateCore aims to stay out of your way.

⚙️ Core Features

  • Readable config-first scheduling (XML)
  • Plug-and-play `IScheduledTask` interface model
  • Custom intervals, daily, weekly
  • Retry policies, error handling hooks, and testing helpers
  • Runs inside any .NET Core Windows Service or Console App

📦 Tech Stack

Developed in C# using .NET and hosted as a lightweight background engine. UI site built with TailwindCSS and GitHub Pages.

🧠 Supported Schedule Types

  • Immediate — Runs once on service start
  • EveryXMinutes — Custom intervals like 5, 15, 30 mins
  • EveryXHours — Run every 2, 4, 6 hours
  • Hourly — Once per hour at a given minute
  • Daily, Weekly, Monthly — Classic patterns
  • TwiceDaily, Weekends, Quarterly — Extended options
